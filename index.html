OPENCLAW CONSOLE

把「小德」做成可以交给老板使用的中控台

早期小德解决的是“自动生成公众号内容”。这次迭代解决的是“系统怎么长期跑、谁来审批、失败怎么发现、产物在哪里、数据库怎么维护”。 我基于 React + Express 做了本地控制台,把 OpenClaw 任务、审批入口、内容数据库、桌面产物、备用通知和诊断日志收进同一个操作界面。

工作流控制启动 Mode A 日报、批准 Mode B 论文速览、追踪候选与写稿阶段。
内容数据库管理已发库、备选池、论文池,支持查重、字段修复和 JSON/SQLite 同步。
小德运维检查 Gateway、RPC、通知通道、OpenClaw 任务和网络入口,支持重启与自诊断。
发布中心查看 HTML 产物、桌面文件、失败任务和修复入口,让交付结果可追踪。
